
# Kafka 연습장

***

아래는 Kafka를 쉽게 이해하기 위해서 주요 언어를 정리해 둔 것이다.

***

1. 데이터의 생성과 소비 (메시징 주체)
프로듀서 (Producer): 카프카 시스템으로 메시지(데이터)를 생성하고 전송하는 주체입니다.

컨슈머 (Consumer): 카프카에 저장된 데이터를 읽어와서 소비하는 주체입니다.

컨슈머 그룹 (Consumer Group): 하나의 토픽을 구독하는 여러 컨슈머들의 논리적인 묶음입니다. 데이터를 병렬로 빠르게 처리할 수 있게 해주며, 그룹 내 특정 컨슈머에 장애가 발생하면 다른 컨슈머가 작업을 이어받아 처리 안정성을 높입니다.

2. 데이터 저장 및 관리 단위
토픽 (Topic): 데이터가 저장되는 카테고리 또는 주제입니다. 관계형 데이터베이스(RDB)의 '테이블'이나 파일 시스템의 '폴더'와 유사한 개념입니다.

파티션 (Partition): 하나의 토픽을 여러 개로 쪼갠 물리적인 단위입니다. 데이터가 파티션별로 분산 저장되기 때문에 프로듀서와 컨슈머가 병렬로 데이터를 빠르게 읽고 쓸 수 있습니다.

오프셋 (Offset): 파티션 내부에 저장된 각 메시지에 부여되는 고유한 일련번호(위치 값)입니다. 컨슈머는 이 오프셋을 기준으로 자신이 데이터를 어디까지 읽었는지 추적합니다.

3. 서버 및 아키텍처
브로커 (Broker): 카프카 애플리케이션이 설치되어 실행 중인 개별 서버(노드)입니다. 메시지를 실제로 저장하고, 프로듀서나 컨슈머의 요청을 처리하는 역할을 합니다.

클러스터 (Cluster): 대용량 처리를 위해 여러 대의 브로커 서버를 묶어 하나의 거대한 시스템처럼 동작하게 만든 구성입니다. 하나의 브로커가 고장 나도 다른 브로커가 역할을 대신하여 데이터 유실을 방지합니다.

주키퍼 (ZooKeeper) / 크래프트 (KRaft): 카프카 클러스터의 설정 정보(메타데이터)를 관리하고 브로커들의 상태를 조율하는 시스템입니다. 과거에는 주키퍼를 필수적으로 사용했으나, 최근 버전에서는 외부 시스템 의존도를 낮추기 위해 카프카 내부의 자체 메타데이터 관리 시스템인 KRaft(Kafka Raft)를 도입하여 사용하고 있습니다.
